# v1.0.130 — Code quality: reliability, security, and test improvements

Release date: 2026-04-09

## Summary

Internal quality release focused on correctness, security hardening, and test
reliability. No new user-facing features. All changes are backward-compatible.

## Security

- **WebSocket origin validation**: the WebSocket upgrader now enforces that the
  `Origin` header matches the server `Host` by default. Requests from
  unrecognised origins are rejected with 403. Non-browser clients that omit
  `Origin` are unaffected. Previously any webpage could open a WebSocket
  connection to a reachable port.

## Reliability & correctness

- **TTL goroutine context propagation**: `output.NewFile` now accepts a
  `context.Context`. The background TTL goroutine listens on `ctx.Done()` and
  exits cleanly on shutdown, preventing a goroutine leak when the process is
  interrupted before the TTL elapses.

- **TUI readiness signal**: replaced the `waitForUIReady` polling loop (which
  read a hardcoded `/tmp` path in a 25 ms sleep loop) with a proper
  `readyCh chan struct{}` passed into `ui.Run`. The UI closes the channel on
  the first `WindowSizeMsg`, eliminating the race-prone file-based
  synchronisation and the hard-coded world-writable path.

- **Injectable clock in Aggregator**: `Aggregator` exposes a `nowFunc`
  field (default `time.Now`) used in all internal time calls. A
  `newWithClock` test helper allows tests to advance time deterministically
  without sleeping.

- **Error handling for operational writes**: `auditLogger.Log`, `pOutput.Write`,
  `pOutput.Close`, and `f.Close` calls in `runTrace` and `runTraceWithEvents`
  no longer silently discard errors. Failures are now logged as warnings to
  stderr so audit trail gaps and incomplete flushes are visible.

- **Race detector on coverage runs**: `make coverage` now passes `-race` to
  `go test`, ensuring data races are caught in coverage CI pipelines.

- **Pointer receivers on model methods**: `ModelFromAggregator` and related
  methods were converted to pointer receivers for consistency and to support
  readiness signalling.

- **Unused constants removed**: cleaned up dead constants from the UI package.

## Tests

- `TestNewFileAutoExpire_CancelledByContext` — verifies that cancelling the
  context before TTL expiry preserves the file (goroutine exits without
  removing it).

- `TestStream_RejectsDisallowedOrigin` — confirms that a WebSocket upgrade with
  a mismatched `Origin` is rejected.

- `TestStream_AllowsMatchingOrigin` — confirms that a WebSocket upgrade where
  `Origin` matches the server host succeeds.

- `TestRate_UpdatesAfterBurst` and `TestAggregator_RateAfterBurst` — rate
  calculation tests rewritten to use the injectable clock; the previous 600 ms
  `time.Sleep` is eliminated.

- `t.Parallel()` removed project-wide: the calls added no measurable benefit
  for a CLI tool and introduced data races in the `cmd` package due to
  package-level global variables. Tests now run sequentially and reliably.

- Additional unit tests for `RenderLog`, `RenderHelp`, `RenderFiles`,
  `stringsRepeat`, and `helpersFormatCount` UI render helpers.

- Audit logger tests extended with `TestAudit_LogTraceEndWithFileHash`.

## UI enhancements (since v1.0.128)

- Log overlay: pause/resume log fetching with a keypress; improved log table
  layout with visible controls in the dashboard.

- Help overlay: restructured content with clearer sections.

- `RenderFiles` and `RenderLog`: simplified height calculations and improved
  visibility handling.

- `docker-compose`: polling configuration added to the site service.

## Internal / Maintenance

- `go test ./...` passes cleanly with `-race -count=1` across all packages.
- `make lint` reports 0 issues.
- `make coverage` now includes the race detector.

## Migration notes

No breaking changes. If you run the WebSocket server (`--serve`) behind a
browser-based frontend on a non-standard origin, pass the origin explicitly or
configure a reverse proxy that forwards the correct `Origin` / `Host` headers.
