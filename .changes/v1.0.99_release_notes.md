# v1.0.99 — Per-file view, TUI & sidecar diagnostics

Release date: 2026-03-19

Highlights
----------

- Per-file view: identify the most-opened file paths during a trace and surface
  them across TUI, HTTP sidecar API, and the HTML report.
  - TUI: press `f` to open the Top Files overlay (scroll with ↑/↓ or j/k).
  - Sidecar: `GET /api/files?limit=N` returns the top N opened paths.
  - Report: `--report-top-files N` includes the Top Files table in the
    self-contained HTML report.

- Aggregator improvements
  - `fileStats` and `fileStatsByCall` track observed paths and per-syscall
    attribution.
  - `fd->path` mappings to attribute fd-based syscalls to paths where possible.

- TUI & UX
  - `FILE` column shows top observed path for each syscall; overlay with `f`.
  - Rune-aware truncation and sanitisation to avoid terminal corruption from
    control bytes and NULs.
  - Fallback terminal-size detection when a WindowSize message is not delivered
    (useful under `sudo` / re-exec / headless environments).
  - UI diagnostic event logging at `/tmp/stracectl_ui_events.log` to help
    reproduce TUI initialization races.

- Server & diagnostics
  - New endpoint: `GET /api/files` (supports `?limit=N`).
  - WebSocket write deadlines to avoid blocking slow clients.
  - Prometheus `stracectl_ws_clients` gauge for active WebSocket clients.
  - `/debug/goroutines` and pprof handlers added for remote diagnostics.

- Misc
  - `run` / `attach` now start the UI earlier and include a `waitForUIReady`
    helper to reduce terminal initialization races.
  - Unit tests covering path extraction and TopFiles behaviour were added.

Upgrade notes & usage
---------------------

- TUI: press `f` to show the Top Files overlay.
- Sidecar: query `GET /api/files?limit=20` to fetch the most-opened paths.
- Report: pass `--report-top-files N` to include the top-N files in the
  generated HTML report.

Known caveats
-------------

- eBPF: eBPF C sources and BPF generation may require a matching toolchain
  (clang, kernel headers, `bpf2go`). If you build the eBPF-enabled binary,
  follow the existing instructions in `README.md` and `docs/EBPF.md`.
