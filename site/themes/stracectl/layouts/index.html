Three modes,
one tool

01

TUI Mode

Run or attach to any process. The htop-style dashboard updates live every 500 ms — sort, filter, explore details without leaving the terminal.

sudo stracectl run nginx
sudo stracectl attach 1234
02

Sidecar / HTTP API Mode

Add --serve :8080 to replace the TUI with an HTTP server — JSON at /api/syscalls, WebSocket live feed, Prometheus metrics, and a React-free live dashboard at /.

sudo stracectl attach --serve :8080 1234
03

Post-mortem / Stats Mode

Already have a strace -T -o log? Feed it to stracectl stats for the same TUI, HTTP API, or HTML report — no live process needed.

stracectl stats --report out.html trace.log

Two tracing backends —
eBPF and strace

Pick the tracer that best fits your environment: low-overhead kernel events or the universal strace fallback.

🦺

eBPF backend

Kernel-level tracing with minimal overhead. Requires Linux ≥ 5.8 and an eBPF-enabled binary (built with -tags=ebpf). Ideal for production observability.

Learn more →
🧾

strace backend

Classic userspace tracer via the strace binary. Works on systems without eBPF support and requires no special kernel features.

--backend auto|ebpf|strace

Not just counts —
built-in explanations

Press Enter on any row to see the syscall's signature, arguments, errno codes, and diagnostic notes — inline, without leaving the terminal. The same reference is available on every web detail page.

openat I/O alias: open

Open or create a file, returning a file descriptor.

openat(dirfd, pathname, flags, mode) → fd

💡 High ENOENT rates are normal — the dynamic linker probes many paths when loading shared libraries.

connect NET

Initiate a connection on a socket to a target address.

connect(sockfd, addr, addrlen) → 0

💡 ~50% error rate is expected with Happy Eyeballs (RFC 8305) — IPv4 and IPv6 race in parallel; the loser always fails.

futex OTHER

Fast user-space locking — the kernel backing for mutexes and condition variables.

futex(uaddr, op, val, timeout, …) → 0

💡 High syscall counts suggest lock contention — futex only enters the kernel when a thread must actually sleep.

~50 canonical syscalls documented — signatures, arguments, errno codes, and real-world diagnostic notes.
